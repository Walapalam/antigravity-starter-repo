# Code Review Skill

## Description
This skill helps you perform thorough code reviews, identifying potential issues, suggesting improvements, and ensuring code quality standards are met.

## When to Use
- When reviewing pull requests
- Before merging code changes
- When assessing code quality
- When looking for potential bugs or security issues

## How to Use
1. Examine the code changes carefully
2. Check for:
   - Code style and consistency
   - Potential bugs or logic errors
   - Security vulnerabilities
   - Performance concerns
   - Test coverage
   - Documentation completeness
3. Provide constructive feedback
4. Suggest specific improvements with examples

## Best Practices
- Be specific and actionable in your feedback
- Explain the "why" behind your suggestions
- Prioritize critical issues over style preferences
- Acknowledge good practices you observe
- Consider the context and constraints of the project
